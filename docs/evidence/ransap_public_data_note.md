## Source Facts Checked

The public RanSAP repository describes the dataset as storage access patterns for
seven ransomware samples, five benign software samples, extra ransomware
variants, OS variation, and BitLocker-enabled storage conditions.

The repository README states that RanSAP can be cloned from GitHub or downloaded
from Kaggle:

- <https://github.com/manabu-hirano/RanSAP>
- <https://www.kaggle.com/datasets/hiranomanabu/ransap-2022-ransomware-behavioral-features/>

The same README lists the storage CSV schemas used by this repository:

- `ata_read.csv`: UNIX seconds, UNIX nanoseconds, LBA, size.
- `ata_write.csv`: UNIX seconds, UNIX nanoseconds, LBA, size, Entropy #1,
  Entropy #2.

The README says large CSV files may be compressed in zip archives and provides
`unzip-all.sh`.

License and citation requirements are not optional. The README says RanSAP is
available under Creative Commons Attribution-NoDerivatives 4.0 International and
asks users to cite:

- M. Hirano, R. Hodota, R. Kobayashi, "RanSAP: An open dataset of ransomware
  storage access patterns for training machine learning models," Forensic
  Science International: Digital Investigation, Volume 40, 2022.
  <https://doi.org/10.1016/j.fsidi.2021.301314>
- M. Hirano and R. Kobayashi, "Machine Learning Based Ransomware Detection Using
  Storage Access Patterns Obtained From Live-forensic Hypervisor," IOTSMS 2019.
  <https://doi.org/10.1109/IOTSMS48152.2019.8939214>

## Use In This Repository

`analysis/ae_public_eval/run.py` consumes only the storage CSV subset above. It
does not use process names, file paths, signatures, host filesystem metadata, or
memory-access features. Write entropy is treated as optional telemetry and must
be ablated before deployment claims.

Raw RanSAP files should be staged locally under `data/raw/RanSAP/` and excluded
from git. Results promoted as evidence must be generated by script and recorded
in `analysis/ae_public_eval/results/*/manifest.json`.

## Claim Boundary

This note verifies dataset access, schema compatibility, and license handling.
It does not verify model performance, storage-device deployability, MNN
conversion, or 500 KB per-volume detector-data fit.
