\section{妥当性への脅威}

\paragraph{公開データの偏り}
RanSAP/RanSMAP は初期検証に有用だが、ransomware family、OS、volume condition、
benign workload の範囲は限定的である。headline accuracy ではなく、
family holdout、condition holdout、benign workload holdout を使う必要がある。

\paragraph{未知脅威評価の代理性}
教師なし AE は攻撃 label を学習に使わないため未知・変種攻撃に対する余地を持つが、
family holdout は将来の未知脅威そのものではない。held-out family で recall が残っても、
低速暗号化、部分暗号化、正規管理処理に似た攻撃、または観測できない host-side context に
依存する攻撃を保証しない。このため、未知脅威 claim は family/variant 別の失敗分析と
controlled replay による stress test が揃うまで proxy claim に留める。

\paragraph{entropy 依存}
write entropy が高い ransomware は検出しやすい一方、既に暗号化されたボリューム、
圧縮済み benign file、backup、deduplication では entropy が信頼できない。
また、dataset-provided write entropy は embedded collector の cheap telemetry と
同一ではない。metadata-only variant を主結果として扱えるかが鍵になる。

\paragraph{workload intensity の混同}
AE は入力分布全体を再構成するため、ransomware ではなく単に write intensive な
作業を異常とみなす危険がある。これを防ぐには intensity-only baseline と
channel-wise attribution が必要である。

\paragraph{10 秒窓の遅延}
10 秒統計は装置負荷を抑えるが、attack start 直後の小さなシグナルを隠す可能性がある。
time to detect だけでなく、raw trace と攻撃ラベルから後処理で推定した
alert 前の affected LBA fraction を測る。この fraction は検出器の入力特徴ではなく、
検出の遅さがどれだけ被害量に変換されるかを見る評価指標である。

\paragraph{MNN 変換差}
offline framework で良好な AE が、MNN 変換や量子化後に score ordering を保つとは
限らない。MNN parity は研究後半の付録ではなく、実装主張の中心評価項目である。

\paragraph{多数 volume での累積コスト}
volume あたりの detector が軽量でも、ストレージ装置内で多数の volume/namespace に
同時適用すると、入力統計、normalization state、threshold、score history、
inference slot が累積する。単一 volume の 500 KB detector-data fit だけでは
device-fit claim として不十分であり、想定 volume 数での aggregate memory と
CPU scheduling を測る必要がある。共有 runtime/library は 500 KB 予算から除外するが、
総メモリ式では共有項として明示する。
